# JSON
- JavaScript Object Notation
- a lightweight, text-based, data-interchangeable format
- language independent, although closely tied to JavaScript
- widely used to exchange data in the web/cloud environment
- based on sequences on name-value pairs:
  - `"colour": "blue"`
  - can be read or written to file, network connection, or anywhere text data can be stored
- JSON data represents object values
  - based directly on the JavaScript representation of objects as a list of name/value pairs
  - separated by commas
  - bracketed by braces to create a list
  - values have a JSON data type: `string`, `number`, `object`, `array`, `boolean`, `null`
- JSON object lists can be nested
  - each nested list is a name/value pair
- JSON also has arrays
  - the array of lists (objects) is in square brackets
  - each array item separated by a comma

# CSV
- simple text file format used to store tabular data
  - spreadsheet, or database data
- each line in a CSV file corresponds to a ro win the table
- each field in that row (or cell in the table) is separated by a comma
  - other delimiters like semicolons can also be used
- format defined by RFC 4180, published by the IETF, 2005
